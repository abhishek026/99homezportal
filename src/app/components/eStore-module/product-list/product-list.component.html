FAQâ€™S & Help Descriptions.

Add Product
Before creating any product, make sure e-course should be created from Course > E-Courses mapping.
E-courses
  1. Admin can link multiple ecourses with a single product
  2. Product items from all the above selected ecourses will be fetched below.
Description
  1. It describes about the particular product which will be visible to end users.
  2. Keep the description as short as possible, single liner is preferred.
Product sell Limit
  1. If admin wants to sell that particular product on limited end users then they have to give limit.
  2. Number of products which can be sold from the web/app will be as per the defined product sell limit
Actual Price
It is MRP of the product.
Discount
If want to show some %age off on product to students then admin can give discount value.
Final Price
Total amount of the product which student has to pay after discount if applied.
GST
  1. If Products to be sold with GST then request proctur team to enable this.
  2. If GST applied then Final price will be considered as incl of GST only.
Total user purchased product
  1. This option is only to show end users that how many products has been purchased.
  2. Admin can give any value and it will be visible to end users. This is introduced only for marketing purpose.
  3. Eg: If admin give 100 value, then on end users side it will be visible as 100 users have purchased the products and will increase the count as an when student purchase the product.
Tags
If admin want to highlight any product they can select through dropdown.
Is advanced product(Put default enabled)
  1. Enabling this feature published product will be available for editing.
  2. While editing admin can add the product items in published product or change the description, no access for edit the previous content.
Product for- Student
If admin want to show any products to the enrolled students only then he/she has to select students.
Product for- Open user
  1. If admin wants to show any product to the open app users then he/she has to select open app users.
  2. Open app user - Who registered through estore on web portal or white labelled app.
Product items
These are the items which can be added to the product whose data will be fetched on the basis of selected e-course.
Sales date range
This is the date range in which the product can be sold.
Visibility date range
  1. This is the product validity date range. The student can access the product in the defined date range only.
  2. Product visibility start date cannot be prior or same day to sales start date.
Show Duration
This is the product validity duration. The student can access the product in the defined duration only, starting from the purchase date. Duration can be set in number of days.
Copy link
If admin wants to share any particular products then they can share link of that products to end users.
Please note- Admin can't be able to change the product sales date range after publish any product.
